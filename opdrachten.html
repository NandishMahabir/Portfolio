Qern Project WPL2

Beschrijving

Tijdens Werkplekleren 2 werkten we aan een groepsproject voor het externe bedrijf Qern. Het doel van dit project was om een volledige rebranding uit te voeren en een nieuwe website met bijhorend admin dashboard te ontwikkelen. Het project werd uitgevoerd door een multidisciplinair team bestaande uit DVO-studenten en PRO-studenten . De opdracht bestond uit het ontwerpen en bouwen van een moderne, gebruiksvriendelijke website die de nieuwe identiteit van Qern zou weerspiegelen. Daarnaast moest er een admin dashboard ontwikkeld worden waarmee het bedrijf hun diesnten en gegevens kon beheren. Het project was opgesplitst in verschillende sprints waarin zowel design- als developmenttaken moesten worden uitgevoerd en op elkaar afgestemd. In een normale situatie zou het team meerdere programmeurs bevatten om de technische implementatie te verdelen. Tijdens het project vielen echter twee programmeurs vroeg weg, waardoor het developmentwerk sterk werd beperkt tot één persoon. Dit had een grote impact op de planning en uitvoering van de technische onderdelen van het project. Ondanks deze uitdaging bleef het team samenwerken aan het eindresultaat, waarbij de focus lag op het opleveren van een functionele website en een admin dashboard.

Bijdrage

Mijn bijdrage binnen dit project lag volledig binnen het developmentgedeelte. Ik was verantwoordelijk voor het opstellen van het datamodel en het opzetten van de database-structuur, zodat alle gegevens correct en efficiënt konden worden opgeslagen en beheerd. Daarnaast heb ik de backend-architectuur uitgewerkt in .NET. Dit hield in dat ik de basisstructuur van de applicatie heb opgezet en de logica van het systeem heb georganiseerd zodat het project schaalbaar en onderhoudbaar bleef. Binnen deze backend heb ik ook de nodige endpoints ontwikkeld met ASP.NET Web API, zodat de frontend en het admin dashboard correct konden communiceren met de server en database. Naast de backend heb ik ook gewerkt aan het admin dashboard in Vue. Hierbij zorgde ik voor de integratie tussen frontend en backend, zodat gebruikers via het dashboard data konden beheren en aanpassen. Dit vereiste dat ik zowel de technische werking van de API als de noden van de gebruikersinterface goed op elkaar moest afstemmen. Door het wegvallen van twee programmeurs werd mijn rol veel uitgebreider dan oorspronkelijk gepland. Hierdoor moest ik sterk inzetten op prioriteiten stellen en zelfstandig beslissingen nemen over de technische opbouw van het project. Niet alle features konden daardoor volledig worden uitgewerkt binnen de beschikbare tijd, waardoor ik keuzes moest maken op basis van wat het belangrijkst was voor het eindresultaat. Deze ervaring heeft mijn technische vaardigheden sterk verdiept, vooral op vlak van backend development, API-ontwikkeling en full-stack integratie. Daarnaast heb ik geleerd om zelfstandiger te werken en beter om te gaan met projectdruk en verantwoordelijkheid binnen een realistische werkomgeving.
